举个最简单的联合体：
```C++
union Test {
    int a;    // 4字节
    char b;   // 1字节
};
```
内存布局是这样的：
```
内存地址：  0  1  2  3
          ┌───────────┐
a 存储：  │  int a    │  4字节
          └───────────┘
          ↑
b 存储：  │ char b │      共享同一起始地址！
```

- 给 a 赋值，这块内存存的是整数
- 给 b 赋值，这块内存存的是字符

你不可能同时存两个值，后赋值的会覆盖先赋值的

1. 为什么要这么设计？
C++ 设计联合体的目的只有一个：
- 节省内存空间
- 在早期内存非常小的设备（单片机、嵌入式）里，有些数据同一时间只会用到一个，没必要给每个成员都分配独立内存。
比如：一个变量要么存整数，要么存浮点数，二选一，用联合体只占最大成员的大小，不浪费空间。
