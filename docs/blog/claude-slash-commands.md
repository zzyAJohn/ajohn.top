Claude Code 有大量斜杠命令，但官方文档散落各处，用的时候经常想不起来。这篇文章把所有命令整理在一起，按用途分类，配上实际使用场景，方便速查。

> 本文基于 Claude Code CLI **v2.1.153**，命令随版本更新可能有变化。用 `/help` 查看你当前版本的完整命令列表。

<!-- more -->

## 一、会话管理

日常使用最频繁的一组命令，管对话、管配置、管状态。

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/help` | 显示帮助信息和可用命令列表 | 刚上手不知道有啥命令时 |
| `/clear` | 清空对话历史，重新开始 | 对话跑偏了，想从头来 |
| `/compact` | 压缩对话上下文，减少 Token 占用 | 对话太长，上下文快满了。支持自定义指令，如 `/compact 只保留架构相关讨论`，决定保留哪些内容 |
| `/usage` | 显示当前会话的 Token 用量和费用 | 想知道这轮对话花了多少钱（旧版本是 `/cost`） |
| `/context` | 显示详细的上下文用量分析 | 查看各部分（系统提示、工具、技能、消息）各占多少 Token，排查上下文满的问题 |
| `/skills` | 列出所有可用的 Skills | 想知道有哪些斜杠命令可以用，包括内置的、项目的、用户的 |
| `/config` | 查看或修改配置 | 改设置，比如主题、模型偏好 |
| `/model` | 切换模型 | 想从 Sonnet 切到 Opus，或反过来 |
| `/status` | 显示当前会话状态和连接信息 | 排查连接问题 |
| `/doctor` | 运行诊断，检查配置是否正常 | Claude Code 行为异常时排查问题 |
| `/diff` | 显示本次会话中修改的 git diff | 看看这轮对话改了哪些文件 |

## 二、认证与权限

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/login` | 登录或切换 Anthropic 账号 | 首次使用或换账号 |
| `/logout` | 退出登录 | 切换环境时 |
| `/permissions` | 查看和管理工具权限 | 想知道哪些工具被允许/禁止了 |

## 三、代码审查与安全

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/code-review` | 审查当前 diff，找 bug 和优化点 | 提交前想让 AI 帮忙 review。加 `--comment` 可以在 PR 中加行内评论，加 `--fix` 直接修复 |
| `/simplify` | 审查 diff 并自动修复（等同于 `/code-review --fix`） | 想快速优化代码，一键搞定 |
| `/security-review` | 对当前分支的变更做安全审查 | 上线前的安全检查 |
| `/review` | 审查 Pull Request | 看别人提的 PR，让 AI 帮忙审 |
| `/pr_comments` | 查看当前分支的 PR 评论 | 想看 reviewer 说了什么 |

## 四、开发工作流

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/run` | 启动并驱动项目应用，看改动是否生效 | 改完代码想跑起来看看效果 |
| `/verify` | 验证代码变更是否真的生效了 | 确认改动不只是编译通过，而是实际工作 |
| `/init` | 初始化项目的 CLAUDE.md 文件 | 新项目第一次用 Claude Code |
| `/memory` | 编辑 CLAUDE.md 记忆文件 | 想更新项目的 AI 记忆 |

|------|------|---------|

## 六、终端与编辑器

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/terminal-setup` | 配置终端集成（如 Shift+Enter 换行） | 首次使用时配置终端 |

## 七、内置 Skills（斜杠命令形式）

这些是 Claude Code 自带的 Skills，通过斜杠命令触发：

| 命令 | 作用 | 使用场景 |
|------|------|---------|
| `/loop` | 按间隔重复执行某个命令，如 `/loop 5m /foo` | 定时轮询部署状态、定期检查等 |
| `/claude-api` | 构建、调试和优化 Claude API 应用 | 开发基于 Anthropic SDK 的项目时 |
| `/fewer-permission-prompts` | 扫描常用操作，自动添加权限白名单减少弹窗 | 被权限确认烦死了 |
| `/keybindings-help` | 自定义键盘快捷键 | 想改快捷键 |
| `/update-config` | 通过 settings.json 配置 Claude Code | 需要改权限、环境变量、hooks 等 |

## 八、自定义命令

除了内置命令，你还可以创建自己的斜杠命令。在以下目录放一个 `.md` 文件即可：

- **项目级**：`.claude/commands/xxx.md` → 命令名为 `/project:xxx`
- **用户级**：`~/.claude/commands/xxx.md` → 命令名为 `/user:xxx`

文件内容就是给 Claude 的指令模板。比如创建 `.claude/commands/deploy.md`：

```markdown
部署当前分支到生产环境：
1. 运行测试套件
2. 构建项目
3. 推送到部署目标
4. 验证部署成功
```

然后在 Claude Code 中输入 `/project:deploy` 就能触发。

## 九、快捷键

除了斜杠命令，Claude Code 还有一套快捷键，日常使用频率很高。

### 输入与编辑

| 快捷键 | 作用 |
|--------|------|
| `Enter` | 发送消息 |
| `Shift+Enter` | 换行（写多行输入时不发送） |
| `Alt+Enter` | 换行（另一个快捷键） |
| `Ctrl+J` | 换行（又一个替代键） |
| `Tab` | 接受自动补全建议（文件路径、命令等） |
| `Ctrl+G` | 打开外部文本编辑器编写长提示词 |
| `↑` / `↓` | 浏览历史输入 |

`Ctrl+G` 特别实用：当你需要写一大段复杂的 prompt 时，按 `Ctrl+G` 会打开你系统配置的编辑器（vim、nano 等），写完保存关闭就自动填入输入框。

### 模式切换

| 快捷键 | 作用 |
|--------|------|
| `Shift+Tab` | 在三种模式间切换 |

三种模式：

- **Normal**（默认）：Claude 在修改文件或执行命令前会征求你同意
- **Plan**：Claude 只分析和规划，不做任何修改（只读模式）
- **Auto**：Claude 自动执行，不弹确认框

### 会话控制

| 快捷键 | 作用 |
|--------|------|
| `Escape` | 中断 Claude 当前的回复生成 |
| `Ctrl+C` | 取消当前输入或退出 Claude Code |
| `Ctrl+D` | 退出 Claude Code |
| `Ctrl+L` | 清屏 |

### 特殊前缀

输入时还有两个特殊前缀可以直接操作：

| 前缀 | 作用 | 示例 |
|------|------|------|
| `@` | 引用文件或目录作为上下文 | `@src/index.ts 帮我看看这个文件` |
| `#` | 将内容写入 CLAUDE.md 记忆文件 | `# 项目使用 pnpm，不用 npm` |

`@` 前缀不用手动复制文件内容，直接 `@` 文件路径就能把文件塞进上下文。`#` 前缀更省事，一句话就能更新项目的 AI 记忆。

### 自定义快捷键

Claude Code 支持通过 `~/.claude/keybindings.json` 自定义快捷键，可以定义组合键（chord sequences）绑定自定义操作或斜杠命令。用 `/keybindings-help` 可以查看配置方法。

## 快速发现命令

在 Claude Code 会话中输入 `/help` 可以查看所有可用命令。也可以输入 `/` 然后按 Tab 或 `?` 浏览命令列表。
