>建造者模式（Builder），将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。



::: center
@startuml


together {
    class Director {
        +Construct()
    }

    abstract class Bulider {

        +BuildPart()
    }
}


class ConcreateBulider {

  +BuildPart()
  +GetResult()
}

class Product {

}

note top of Bulider : Builder 是为创建一个 Product \n对象的各个部件指定的抽象接口


note "具体产品" as N1
note "具体建造者，实现 Builder 接口，构造和装配各个部件" as N2
note "指挥者，是构建一个使用 Builder 接口的对象" as N3

Product .. N1
ConcreateBulider .. N2
Director .. N3


Director o--> Bulider
Bulider <|-- ConcreateBulider
ConcreateBulider ..> Product


@enduml
